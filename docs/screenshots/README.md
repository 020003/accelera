# Screenshot Instructions

Please take the following screenshots and save them in this directory:

## Required Screenshots

1. **dashboard-overview.png**
   - Full dashboard view showing the Overview tab
   - Should display multiple hosts with GPU and Ollama metrics
   - Include the header showing connection status

2. **overview-tab.png**
   - The Overview tab content
   - Show aggregated statistics cards
   - Include the multi-host overview section

3. **host-tab.png**
   - Individual host tab view
   - Display GPU cards with metrics
   - Show the tabbed interface with GPUs and AI Models tabs

4. **ollama-models.png**
   - The AI Models tab for a host with Ollama
   - Show the model cards with sizes
   - Include the Ollama detection indicator

5. **ollama-metrics.png**
   - The Ollama metrics cards
   - Show AI Models count, Model Storage, Requests, and Performance cards

6. **ollama-tab.png**
   - Full view of the AI Models tab
   - Display all available models in grid layout

7. **settings-tab.png**
   - Settings tab showing host management
   - Include the add host form
   - Show refresh interval and energy rate settings

## How to Take Screenshots

### Option 1: Browser Screenshot
1. Open the dashboard at http://localhost:8080
2. Press F12 to open Developer Tools
3. Press Ctrl+Shift+P (or Cmd+Shift+P on Mac)
4. Type "screenshot" and select "Capture full size screenshot"
5. Save with the appropriate filename

### Option 2: OS Screenshot Tools
- **Windows**: Win+Shift+S for Snipping Tool
- **Mac**: Cmd+Shift+4 for area selection
- **Linux**: Use screenshot tool or `gnome-screenshot -a`

### Option 3: Browser Extensions
- Use extensions like "Full Page Screen Capture" or "Awesome Screenshot"

## Image Guidelines

- **Format**: PNG preferred for quality
- **Resolution**: At least 1920x1080 for full dashboard views
- **Content**: Ensure sensitive information is not visible
- **Consistency**: Use the same browser zoom level for all screenshots
- **Theme**: Preferably use dark mode for better visibility

## Post-Processing (Optional)

- Add red boxes to highlight key features
- Blur any sensitive host information if needed
- Crop to focus on relevant sections
- Optimize file size with tools like TinyPNG
